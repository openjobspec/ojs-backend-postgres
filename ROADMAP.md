# Roadmap

This roadmap describes the current priorities for improving ojs-backend-postgres.

## Near-term priorities

1. **Production safety and observability**
   - Keep metrics cardinality bounded and improve operational visibility.
   - Expand runtime-focused tests for scheduler and PostgreSQL backend behavior.

2. **Project quality and contributor workflow**
   - Maintain issue/PR templates and contribution hygiene.
   - Keep README and changelog aligned with repository state.

3. **Release readiness**
   - Add repeatable release/versioning workflow.
   - Improve public documentation for deployment and operations.

## Mid-term priorities

1. **Developer adoption**
   - Provide worker implementation examples in Go and other common languages.
   - Improve API discoverability with machine-readable API docs.

2. **Operations at scale**
   - Add deployment guidance for production environments.
   - Publish performance benchmarks and tuning guidance.

## Long-term priorities

1. **Ecosystem integrations**
   - Improve interoperability with event-driven systems and external platforms.

2. **Advanced enterprise features**
   - Expand authentication/authorization options and operational controls.
